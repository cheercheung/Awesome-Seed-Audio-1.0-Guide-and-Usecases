# Security Policy

## Supported Scope

This repository is a curated Seed-Audio 1.0 usecase collection. It does not ship runtime infrastructure, credentials, or generated audio payloads.

Security reports should focus on repository content risks, including exposed secrets, unsafe copy-paste commands, malicious links, or misleading instructions.

## Reporting A Vulnerability

Open a private security advisory on GitHub if available. If private advisories are unavailable, open an issue with the minimal public detail needed to route the report, then avoid posting secrets or exploit instructions publicly.

Please include:

- affected file or link
- risk summary
- suggested fix, if known

Maintainers should acknowledge valid reports within 7 days and remove or correct unsafe content as soon as practical.
