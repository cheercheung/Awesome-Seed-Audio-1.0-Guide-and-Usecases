# Contributing to awesome-seed-audio-1.0-usecases

Thank you for helping improve this Seed-Audio 1.0 usecase repository.

## What We Accept

We accept public, source-linked Seed-Audio 1.0 cases with concrete evidence:

- narration, dubbing, voiceover, or audiobook workflows
- audio-first video generation workflows
- multi-character audio drama or scene-generation demos
- reference voice, character voice, or image-guided audio tests
- provider, API, tool, or assistant integrations
- pricing, cost, latency, quality, or limitation observations

## Required Case Metadata

Every proposed case must include:

- public source URL
- creator handle and profile URL
- concise title
- usage takeaway
- source-grounded notes
- type: Demo, Tutorial, Evaluation, Integration, Benchmark, or Limit
- publication date in `YYYY-MM-DD`

## Repository Policy

Raw source exports and internal curation datasets are not stored in public release content. Public traceability is carried by the source links in each case heading.

## Formatting

Use the existing README format:

```md
<a id="case-x"></a>
### Case X: [Title](source-link) (by [@author](author-link))

**Usage takeaway.**

Source-grounded notes.

Type: Demo | Date: YYYY-MM-DD
```

Do not invent prompts, benchmark scores, audio quality claims, pricing, or unsupported results.
